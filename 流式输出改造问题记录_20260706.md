## 二、核心架构演进

### V30 基础流式架构
**核心设计：**
- 流式期间只往 `fullYearlyTextRef.current` 蓄水
- 17张卡片保持骨架屏呼吸灯
- `[DONE]` 信号到达后，调用 `parseYearlyReportV24` 一次性切片分配到17张卡片

**代码位置：** `generateWealthReport` 函数内
```typescript
// 流式期间只蓄水
fullYearlyTextRef.current = (fullYearlyTextRef.current || '') + newChunk;

// [DONE]后总装
if (dataStr === '[DONE]') {
  const finalMap = parseYearlyReportV24(fullText, trueZodiac);
  setYearlyCardData(finalMap);
  setYearlyCardsReady(true);
}
```

**问题：** 17张卡片平铺太长，用户需要频繁滚动

## 三、当前问题（V39仍卡顿）

### 现象
点击"生成财富年报"后：
1. 日志显示：`⚠️ Supabase 无数据 → fallback 到 API`
2. 然后无后续日志
3. 页面卡住，无流式输出

### 可能原因分析

#### 原因1：Railway部署未生效
- 新域名 `kindredsouls-production-3ef1.up.railway.app` 可能仍跑旧代码
- 需要手动Redeploy才能拉取最新commit

#### 原因2：流式端点异常
- `/api/wealth-oracle/stream` 可能返回非流式响应
- 或返回格式不符合预期，导致解析卡住

#### 原因3：前端状态未正确初始化
- `fullYearlyTextRef` 初始为空
- 但`yearlyTextTick`刷新时，`sacredContent`计算可能有问题

#### 原因4：后端API问题
- DeepSeek API调用失败
- 或stream响应格式错误
